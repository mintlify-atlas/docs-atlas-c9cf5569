The E-Dokan storefront is composed of two layers of components:

- **Feature components** — higher-level components that implement specific product or page concerns (navigation, product listings, galleries, etc.).
- **UI primitives** — low-level, reusable building blocks consumed by feature components and pages.

All components are written in TypeScript and styled with Tailwind CSS. Shared prop interfaces live in `types/types.ts`.

## The `cn()` utility

All components that merge Tailwind classes use the `cn()` helper from `lib/utils.ts`:

```typescript
import { type ClassValue, clsx } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

`clsx` conditionally joins class name strings and arrays; `tailwind-merge` then deduplicates conflicting Tailwind utilities (e.g. two `px-*` values) so the last one wins. Pass any combination of strings, arrays, or objects and receive a single, conflict-free class string.

```tsx
import { cn } from "@/lib/utils";

// Conditionally apply an active style
<Link className={cn("text-sm font-medium", isActive && "text-black")} />

// Override a default class from the outside
<Button className={cn("px-2 py-1")} /> // px-2 wins over the default px-5
```
