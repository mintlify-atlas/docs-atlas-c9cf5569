<Note>
  E-Dokan requires a compatible backend API. Set the `NEXT_PUBLIC_API_URL` environment variable to your API's base URL before starting. Without this, all data-fetching will fail.
</Note>

## Prerequisites

- [Node.js](https://nodejs.org/) 18 or later
- npm, yarn, or pnpm
- A running backend API compatible with E-Dokan (providing products, categories, billboards, sizes, and colors endpoints)

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/hempun10/edokan-store.git
    cd edokan-store
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>
      ```bash npm
      npm install
      ```
      ```bash yarn
      yarn install
      ```
      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Configure environment variables">
    Create a `.env.local` file in the project root:

    ```bash .env.local
    NEXT_PUBLIC_API_URL=https://your-api-url.com/api/your-store-id
    ```

    Replace the value with your actual backend API URL. See [Configuration](/configuration) for details on what this URL must provide.
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    The development server starts on [http://localhost:3000](http://localhost:3000).
  </Step>

  <Step title="Open the storefront">
    Visit [http://localhost:3000](http://localhost:3000) in your browser. You should see:

    - A **billboard hero** image at the top of the homepage
    - A **Featured Products** grid showing products with `isFeatured: true` from your API
    - A **navigation bar** with your store name (E-DOKAN) and category links
  </Step>
</Steps>

## Available scripts

| Script | Command | Description |
|---|---|---|
| Development | `npm run dev` | Starts Next.js dev server with hot reload |
| Build | `npm run build` | Compiles the production build |
| Start | `npm start` | Runs the compiled production build |
| Lint | `npm run lint` | Runs ESLint checks |
